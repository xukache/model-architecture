RAG系统类型

Naive RAG

基础架构

最基本的RAG实现方式，通过向量相似度检索相关文档片段，并将其作为上下文提供给大语言模型，实现知识增强的问答系统。

文档分块 向量检索 上下文融合

Structured RAG

结构化增强

利用结构化数据（如表格、JSON）的RAG系统，通过字段映射和模板生成，提供更精确的知识检索和应答能力。

结构化数据 模板生成 字段映射
了解Structured RAG

Graph RAG

图谱增强

结合知识图谱的RAG系统，通过实体关系推理和图谱遍历，提供更丰富的知识上下文和推理能力。

图谱遍历 关系推理 实体链接
了解Graph RAG

Multi-Vector RAG

多向量检索

使用多个向量表示同一文档片段的不同方面，通过多维度检索提高相关性匹配的准确度。

多维表示 混合检索 重排序
了解Multi-Vector RAG

Agentic RAG

智能代理

将RAG与Agent技术结合，通过规划、推理和工具调用，实现更复杂的知识检索和任务处理能力。

工具调用 任务规划 智能推理
了解Agentic RAG

Adaptive RAG

自适应增强

具备上下文学习和策略调整能力的RAG系统，能够根据查询特点和历史交互动态优化检索策略。
